# Reference · DSP 方法菜单

Yeh 定算法时查这本:**每个 block 该用哪种方法、过采样几倍、怎么防混叠。**
原则:**能便宜就别贵**。线性用滤波器,无记忆非线性用查表,有记忆非线性才上迭代。

## 目录

2. 线性 block:双线性变换(bilinear transform)

5. 牛顿迭代(解非线性平衡点)
6. 过采样 + 抗混叠(非线性的命门)

## 1. 决策树(每个 block 先走一遍)

```
这个 block 是线性还是非线性?(问 Merlin)
│
├─ 线性(滤波/缓冲) ──────────────► 用【双线性变换】做成滤波器。便宜、稳、不混叠。完。
│
└─ 非线性(削波/管子)
   │  它带记忆吗?(块里有电容/电感参与非线性反馈吗?)
   │
   ├─ 不带(纯电阻性削波) ─────────► 用【查表 waveshaper】+ 【过采样】。最省事。
   │
   └─ 带(削波和电容耦合,如真 TS/胆机级)
      │  电路简单、元件少?
      ├─ 是 ──► 【WDF】(结构清晰,天生稳定)+ 迭代 + 过采样
      └─ 否/有复杂反馈 ──► 【State-Space】+ 牛顿迭代 + 过采样
```

> 90% 的踏板可以用"线性滤波器 + 无记忆查表削波"搭出来,又快又稳。只有追求胆机级动态 / 复杂反馈时才需要 WDF/state-space。**先用便宜方案做出能听的版本,不够真再升级。**

## 2. 线性 block:双线性变换

**干什么**:把电路的截止频率 → 数字滤波器系数。
**怎么用**:
1. 从电路算连续域的截止频率(如 RC:`f = 1/(2πRC)`,用具体数字)。
2. 套双线性变换的标准公式得到系数(有现成公式表 / RBJ Audio EQ Cookbook,直接查,**不用推导**)。
3. 一阶(RC)或二阶(biquad)按需要选。
**注意**:高频有"频率翘曲(warping)",接近奈奎斯特时截止点会偏——需要时做 pre-warp 校正(公式里加一步)。
**给 Leo**:这一步是"把电路语言(电阻电容值)翻译成代码语言(几个系数)"的标准翻译表,查表照抄即可,不涉及你要理解的数学。

## 3. 无记忆非线性:波形整形查表(waveshaper LUT)

**干什么**:一条固定的"输入→输出"曲线(削波形状),预先烘成表,运行时查。
**怎么用**:
1. 曲线形状来自物理(二极管方程 / tanh 近似 / 实测转移曲线)——**不是随手画的好看曲线**(铁律一)。
2. 离线在 [-N, +N] 范围采样成一张表(如 4096 点),运行时查表 + 线性插值。
**⚠ 命门**:查表削波**一定要包过采样**(见第 6 节),否则刺耳混叠。
**给 Leo**:烘焙。与其每帧实时算那条曲线,不如离线烘成查询表,和你烘 lightmap 一个道理。

## 4. 有记忆非线性:WDF vs State-Space

当削波和电容耦合(输出依赖历史)时,每帧要解方程。两种主流框架:

**WDF(Wave Digital Filters)**
- 把每个元件(电阻/电容/二极管)变成一个"散射结点",像搭积木一样连起来。
- 优点:结构清晰、模块化、**天生数值稳定**、换元件值容易。
- 适合:元件不太多、拓扑清楚的踏板级。David Yeh 的论文和 Kurt Werner 的工作是标准参考。

**State-Space**
- 把整个电路写成一组状态变量(电容电压们)的更新规则,整体一次解。
- 优点:能处理**复杂反馈 / 多非线性耦合**;适合更大的电路(整台 amp preamp)。
- 代价:推导更重、加非线性要配牛顿迭代。

**怎么选**:简单清楚 → WDF;复杂有反馈 → state-space。拿不准先 WDF(更不容易炸)。
**给 Leo**:WDF = 用标准积木块拼电路,每块自带说明书,拼错了也不容易崩;state-space = 把整个系统列成一张"谁影响谁"的更新表,一次全解。两者都由本 lab 产出代码,你不用手推。

## 5. 牛顿迭代(解非线性平衡点)

**干什么**:非线性方程解不开(输出反过来影响输入)→ 猜一个值,看差多少,往反方向修,重复几次收敛。
**怎么用(工程要点)**:
- 初值用**上一帧的解**(信号连续,通常 1–3 次就收敛)。
- **必设迭代上限**(如 8 次)+ 收敛判据(残差 < 1e-7)。
- **必设发散兜底**:输入爆表时可能不收敛 → 超上限就退回上一帧值 / 做 clamp,**绝不允许死循环**(会 xrun)。
**给 Leo**:IK 解算器。一帧内迭代几次把骨骼逼近目标姿势;这里是把电压逼近平衡点。给它个迭代上限,别让它一帧内算到天荒地老。

## 6. 过采样 + 抗混叠(非线性的命门)⚠

**为什么**:任何非线性都会造出新的高频谐波;超过奈奎斯特(采样率一半)的那些会"折叠"回来变成**不和谐的杂音(混叠)**——这就是"数字味 / 廉价插件味"的头号来源。
**做法**:非线性那一段**临时升采样**(如 8×)→ 在高采样率下跑非线性 → **降采样**回来,升/降都配抗混叠滤波器。
**倍数经验起点**:
- 软削波(overdrive、胆机软过载):**4×** 起
- 硬削波 / distortion:**8×**
- fuzz / 极硬钳位:**16×** 甚至更高(谐波极宽)
**怎么定够不够**:交测量台,送单音看谐波谱——**出现非基频整数倍的杂散尖峰 = 还不够,加倍**。
**给 Leo**:super-sampling 抗锯齿。画面细节比像素细就出鬼影 → 先高分辨率算再降采样。声音一样:谐波比奈奎斯特高就折回来变杂音 → 先高采样率算再降回来。**混叠 = 声音里的锯齿。**

---

## 7. ADAA(抗混叠的省事替代)

**是什么**:Antiderivative Anti-Aliasing。对**无记忆**削波,不靠暴力过采样,而是用一点数学技巧直接压掉大部分混叠,**比 8× 过采样省 CPU**。
**何时用**:无记忆 waveshaper 且想省算力时。有记忆的非线性用不了 ADAA,还得老实过采样。
**代价**:实现比查表复杂一点,且对某些曲线要处理数值细节。
**给 Leo**:一个"聪明版抗锯齿",用数学技巧代替暴力超采,省性能。能用则用,用不了就回到第 6 节的过采样。

## 一页速查

| block 类型 | 方法 | 过采样 | 谁验证够不够 |
|---|---|---|---|
| 线性(滤波/缓冲) | 双线性变换 → biquad | 不用 | — |
| 无记忆削波 | 查表 waveshaper(或 ADAA) | 4–16× | 测量台谐波谱 |
| 有记忆削波(简单) | WDF + 牛顿 | 4–16× | 测量台谐波谱 |
| 有记忆削波(复杂/反馈) | State-space + 牛顿 | 4–16× | 测量台谐波谱 |
| cab/喇叭 | IR 卷积 | 不用 | 测量台频响 |
| delay/mod/reverb | 环形缓冲 + 插值 | 不用 | 听感为主 |

> 铁律一提醒:上表每个方法的**参数(系数、曲线、二极管参数)都必须能追回原理图 / 数据手册 / 一条公式**。凑出来的常数一律不合法。
